# CLAUDE.md - rdkitcffi Project Guide

## Project Overview

**rdkitcffi** is a Rust wrapper for the [RDKit](https://www.rdkit.org/) cheminformatics library. It provides safe Rust bindings to RDKit's C Foreign Function Interface (CFFI), enabling molecular manipulation, descriptor calculation, and cheminformatics operations in Rust.

- **Repository**: https://github.com/chrissly31415/rdkitcffi
- **License**: See LICENSE file
- **Supported Platforms**: Linux, Windows, macOS (Apple Silicon/ARM64)
- **Rust Edition**: 2018

## Architecture

### Core Components

1. **Molecule Struct** (`src/lib.rs`)
   - Primary interface for molecular operations
   - Contains raw C pointers (`pkl_mol`, `pkl_size`)
   - **NOT thread-safe** (not `Send` or `Sync`) - each thread must create its own molecules
   - Implements `Drop` trait for automatic memory cleanup

2. **JSON Module** (`src/json.rs`)
   - CommonChem JSON format support
   - Structured types: `JsonBase`, `JsonMolecule`, `JsonAtom`, `JsonBond`, `JsonConformer`
   - Used for serialization/deserialization

3. **Bindings** (`src/bindings.rs`)
   - Auto-generated by bindgen from `include/cffiwrapper.h`
   - Raw FFI function declarations
   - Regenerated during build

4. **Build System** (`build.rs`)
   - Downloads pre-built RDKit libraries from GitHub releases
   - Fallback: builds RDKit from source (Linux only, Release_2021_09 branch)
   - Generates Rust bindings via bindgen
   - Handles platform-specific linking (.so for Linux, .dll/.lib for Windows, .dylib for macOS)

## Key Functionality

### Molecule Operations

- **Construction**: `Molecule::new(smiles)`, `Molecule::new_with_args(input, json_args)`
- **Conversions**: SMILES, SMARTS, InChI, InChIKey, molblock, JSON
- **Modifications**: `add_hs()`, `remove_all_hs()`, `neutralize()`, `normalize()`, `cleanup()`
- **Standardization**: `canonical_tautomer()`, `fragment_parent()`, `charge_parent()`
- **Coordinates**: `set_2d_coords()`, `set_3d_coords()`, `get_coords()`
- **Substructure**: `get_substruct_match()`, `get_substruct_matches()`
- **Descriptors**: `get_descriptors_as_dict()` returns HashMap<String, f32>
- **Fingerprints**: Morgan, RDKit, Pattern (as strings or byte vectors)
- **Visualization**: `get_svg()` generates SVG images

### File I/O

- **SMILES**: `read_smifile()`, `read_smifile_unwrap()`
- **SDF**: `read_sdfile()`, `read_sdfile_unwrap()`, `SDIterator`
- Returns `Vec<Option<Molecule>>` - None for invalid molecules
- `_unwrap` variants filter out None values

### Property System (TODO)

Property functions (`has_prop`, `get_prop`, `set_prop`, `clear_prop`, `keep_props`) are defined in the C header but **currently commented out** in the Rust wrapper. These will be enabled when available in the RDKit CFFI library.

## Build Process

### Automatic Build

1. Run `cargo build` - build.rs automatically:
   - Downloads pre-built RDKit libraries from GitHub releases:
     - Linux: `rdkitcffi_linux.tar.gz` (tag: `rdkit-latest`)
     - Windows: `rdkitcffi_windows_vs.zip` (tag: `rdkit-windows-vs-latest`)
     - macOS: `rdkitcffi_macos.tar.gz` (tag: `rdkit-macos-latest`)
   - Extracts to `rdkitcffi_linux/linux-64/`, `rdkitcffi_windows/windows-64/`, or `rdkitcffi_macos/macos-64/`
   - Creates symlinks (Linux and macOS only)
   - Generates bindings from `include/cffiwrapper.h`

### Fallback Build (Linux Only)

If download fails, build.rs attempts to:
1. Clone RDKit repository (Release_2021_09 branch)
2. Build with CMake (CFFI support enabled)
3. Copy shared libraries to local directory

### Running Tests

```bash
cargo test --lib          # Run library tests
cargo test --all --verbose # Run all tests
```

**Important**: Set library path environment variable for your platform:
- Linux: `export LD_LIBRARY_PATH=$LD_LIBRARY_PATH:./rdkitcffi_linux/linux-64`
- macOS: `export DYLD_LIBRARY_PATH=$DYLD_LIBRARY_PATH:./rdkitcffi_macos/macos-64`
- Windows: DLLs are automatically copied to target directories

## Development Guidelines

### Memory Safety

- All FFI calls are wrapped in `unsafe` blocks
- `Molecule::Drop` ensures cleanup of C-allocated memory
- Always use `free_ptr()` for RDKit-allocated strings
- Use `libc::free()` for libc-allocated memory (e.g., size_t pointers)

### Error Handling

- Constructor returns `Option<Molecule>` - None indicates invalid input
- RDKit errors typically manifest as null pointers
- Most methods panic on null pointers (except constructors)

### JSON Arguments

Many methods accept `json_info: &str` for optional parameters:
```rust
let json_args = r#"{"removeHs":false,"canonical":false}"#;
let mol = Molecule::new_with_args("c1cc(O[H])ccc1", json_args).unwrap();
```

Pass empty string `""` for defaults.

### Thread Safety

The `Molecule` struct is **NOT** thread-safe (no `Send`/`Sync`). For parallel processing:
- Create separate `Molecule` instances in each thread
- RDKit CFFI itself is thread-safe when molecules don't share state

## Common Patterns

### Basic Molecule Creation
```rust
let mol = Molecule::new("OCCO").unwrap();
let smiles = mol.get_smiles("");
```

### With JSON Arguments
```rust
let json = r#"{"canonical":false}"#;
let mol = Molecule::new_with_args("c1ccccc1", json).unwrap();
```

### Reading Files
```rust
let mols: Vec<Molecule> = read_smifile_unwrap("data/test.smi");
```

### Computing Descriptors
```rust
let mol = Molecule::new("CCCN").unwrap();
let desc = mol.get_descriptors_as_dict();
let logp = desc.get("CrippenClogP").unwrap();
```

### 3D Coordinate Generation
```rust
let mut mol = Molecule::new("CO").unwrap();
mol.add_hs();
mol.set_3d_coords("");
let coords: Vec<Vec<f32>> = mol.get_coords();
```

### Substructure Search
```rust
let mol = Molecule::new("c1ccccc1O").unwrap();
let query = Molecule::get_qmol("cO", "").unwrap();
let match_json = mol.get_substruct_match(&query, "");
```
